Initialization - Javascript

Once you include the AngularJS script in your page, create the descriptor of your first application in this way:

angular.module("MyFirstApplication", ["ng"]);

MyFirstApplication is the name that you want in your application.
The second parameter is an array of applications that your app depends, in this case Angular expose ng as the most basic application; if you don't want to extend an application, just send an empty array [].

AngularJS - Digitas - Bosz
Initialization - Storage

You'll like to store the application in a variable, to keep the reference to it in the future.

var myAmazingApp = angular.module...

If you didn't store your application in a variable, you can still reference to it calling to angular.module but without the second parameter.

angular.module("MyFirstApplication")

That will search for the application and return it, or throw an error if it doesn't exists.

AngularJS - Digitas - Bosz
Initialization - Automatic

You can initialize an Angular application automatically in the page load with an attribute in the element you want to be the base of the application. You can use several attributes like: ng:app, ng-app, x-ng-app, data-ng-app

<element ng-app="MyFirstApplication">
    ...
</element>

Note that your application should be declared before been initialized.

AngularJS - Digitas - Bosz
Initialization - Manual

Also, you can initialize it manually, but this need to be in the page load or after that.

window.addEventListener("load", function (){
    angular.bootstrap(element, ["MyFirstApplication"]);
}, false);

In this case, the bootstrap method takes two parameters, the element that will be the application's root, and the second is an array of application's names to initialize in that element.

AngularJS - Digitas - Bosz
Initialization - Uses

If, for some reason, you need to wait for some asynchronous resource to be loaded, your Angular application should wait for some user data or you don't have code access to the tag you want to be the root element, your best choice is to initialize it manually, in the rest of the cases, the best approach is to use the automatic initialization.

Controller - Introduction

The Angular Controller is a place where you put the logic that controls the view (template). You can have more than one controller, but alway one will be the father of the other, you cannot have controllers as brothers.

In the Controller Invocation is where we told Angular which extra code we'll need, but we'll talk about later.

Each controller has it's own Angular Scope so, let's take a look at the Scope first

AngularJS - Digitas - Bosz
Scope - Introduction

The Angular Scope is like the javascript scope, it's just the space of values that you have available in runtime to read and/or write; the main difference is that your scope is always in synchronization with your templates.

An Angular Scope in pure javascript is just a json object, so you can access by dot notation like $scope.property; or by square brackets like $scope["property"];

AngularJS - Digitas - Bosz
Controller - Scope

To initialize a controller you just need to do this:

myAngularApp.controller("NameOfTheController", function ($scope){
    // create variables in the scope here
});

You create a controller with a controller function of your application reference. It takes two parameters, the name of your controller, and a function or an array*. The function is the constructor of the controller, and the parameters it takes are spetial.

In this case the parameter name is $scope, but this is not a random name, Angular will search the object you're requesting to, and call this function with that parameter

(*) We'll talk about the array later.
AngularJS - Digitas - Bosz
Scope - Globally

The $scope object is allergic to the global variables, please don't do this:

$scope.window = window;

The why is because Angular need to control exactly what the $scope stores, and if you do that, the window object is changing continuously, and Angular will no see those changes. There's other way to do this, but we'll talk about them in other lesson.

To edit the $scope or what it stores, you need to be inside of an Angular Function.

AngularJS - Digitas - Bosz
Controller - Implementation

Once declared your controller, you can use it in the html, like this:

<element ng-controller="NameOfTheController">
    ... use normal HTML here, and make templates that use the scope ...
</element>

Inside the tag with the controller, you can read (and write) the scope of your controller and it's parents, wait, WHAT? Yes, you can have a ng-controller inside of an ng-controller, in the child controller, you can use both scopes, but in the parent, you just can use it's own scope.

You can use scope in the templates in this way:

myAngularApp.controller("NameOfTheController", function ($scope){

Also remember that you can perform operations in template, so you can do things like {{myWord}} {{{"true":"", "false":"doesn't"}[myWord.length > 0]}} has mode than 1 letter.

Scope - Data Binding

One of the amazing parts of Angular, is the data-binding, that means that if you change something in you Scope in the javascript, it will be immediately changed in your template, and if you change something in your template, it'll be available immediately in your javascript.

Let's just reveal some Angular components to you, so you can have a more clear idea of this. In any input element, you can write the attribute ng-model, and the value will create or write an variable in your current $scope, and store the value of the input.

<input type="text" ng-model="myUserInput">

Services

The Angular Services are abstracted logic that not control the logic business, their propose are to hold on some constant code while the entire application, and to provide access to common calls or global attributes.

Did you remember that use global variables with the scope is not a recommendation?
Well, if you need to access in the controller to some specific javascript code, the best approach is to create an Angular Service.

AngularJS - Digitas - Bosz
Services - Declaration

To create an Angular Service call to the factory function of your angular application

myAngularApp.factory("nameOfTheService", function (){
    ... logic here ...
    return something;
});

The attributes are like in the controller, the name of the service you want to register, and a function (or an array), this function is the constructor of the service, and should return any kind of data.

The Angular Services are singletons instances, that means that they well be executed one at the start of the application, and never again.

AngularJS - Digitas - Bosz
Services - Implementation

To implement the creation of a Service, Angular created the dependency manager, to just push in your javascript scope what you exactly need for each logic.

myAmazingApp.controller("myAmazingController", function ($scope, MyAmazingService){
    ... use MyAmazingService here ....
});

In this case, you call the service with the dependency manager, Angular will look for a Service named MyAmazingService and will return the constructor's return value, that's why your Service needs to return something, a number, a string, a function or even an object with functions and values.

A last thing, you can call services inside a service, like in controllers, so you can extend your service with another service ;)

AngularJS - Digitas - Bosz
Dependency Manager

Now that you now something about the Dependency Manager I can tell you why the .controller and the .factory can send a function or an array.

Both are correct, the difference is when you minify your scripts, let's see

myAngularApp.controller("myController", function($scope, MyService){...});
Minified 
a.b("myController",function(c,d){...});

The thing is that Angular will search for a service named c or d, those doesn't exists. You can fix that with this:

myAngularApp.controller("myController", ["$scope", "MyService", function(MyScope, MyServiceWithOtherName){...}]);

You put an array, with all your dependencies as strings and the last index is the function that will be called with those services in the same order. Cool right?
