Initialization - Javascript

Once you include the AngularJS script in your page, create the descriptor of your first aplication in this way:

angular.module("MyFirstApplication", ["ng"]);

MyFirstApplication is the name that you want in your application.
The second parameter is an array of applications that your app depends, in this case Angular expose ng as the most basic application; if you don't want to extend an appication, just send an empty array [].

Initialization - Storage

You'll like to store the application in a variable, to keep the reference to it in the future.

var myAmazingApp = angular.module...

If you didn't store your application in a variable, you can still reference to it calling to angular.module but without the second parameter.

angular.module("MyFirstApplication")

That will search for the application and return it, or throw an error if it doesn;t exists.

Initialization - Automatic

You can initialize an Angular application automaticlly in the page load with an attribute in the element you want to be the base of the application. You can use several attributes like: ng:app, ng-app, x-ng-app, data-ng-app

<element ng-app="MyFirstApplication">
    ...
</element>

Note that your application should be declared before been initialized.

AngularJS - Digitas - Bosz
Initialization - Manual

Also, you can initialize it manually, but this need to be in the page load or after that.

window.addEventListener("load", function (){
    angular.bootstrap(element, ["MyFirstApplication"]);
}, false);

In this case, the bootstrap method takes two parameters, the element that will be the application's root, and the second is an array of application's names to initialize in that element.

AngularJS - Digitas - Bosz
Initialization - Uses

If, for some reason, you need to wait for some asynchronous resource to be loaded, your Angular application should wait for some user data or you don't have code access to the tag you want to be the root element, your best choice is to initialize it manually, in the rest of the cases, the best approach is to use the automatic initialization.

Templates

Angular uses the dom as a live template system, that means that you don't neet to create a new file or declare your template inside a <script> tag any more*.

* You can still doing this if you want.
AngularJS - Digitas - Bosz
Templates - Types
  • You can use templates in the HTML as another kind of tag. 
    <p>Hello, my name is {{name}}!</p>
  • You can storage templates in a <script> tag. 
    <script type="text/ng-template" id="my-template.html">
    <p>Hello, my name is {{name}}!</p>
</script>
  • You can storage templates in a string. 
    var myTemplate = "Hello, my name is {{name}}!";

Templates - Expressions

There's some rules of what you can do inside a Angular template; you can do whatever you do in javascript, except to change the control flow of your code*, you just can use the variables available in your scope** and you can use Angular Filters**.

(*) That means no for or if structures (**) We'll talk about that later.
